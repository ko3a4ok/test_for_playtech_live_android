Test task for Playtech Live Android team

~2 hours

  • Please create single screen application.
  • Screen should contain list of items and button "Update"
  • Each item should be one of types: 'male', 'female' and 'child'
  • Every items have attribute 'type', 'name' and 'color_bg'
  • Item of 'male' type has also attributes 'car' and 'childrenCount'
  • Item of 'female' type has also attribute 'children'(list)
  • Item of 'child' type has also attributes 'father' and 'mother'
  • Every items should has solid background which is filled by color from 'color_bg'
  • 'Male' item:
    • should contain blue line in left side
    • 'car' should be located at the center of cell, with big font size and bold stye
    • 'name' is located at top right corner
    • 'childrenCount' is located at bottom right corner
  • 'Child' item:
    • should contain yellow line at top
    • 'mother' is located at top right corner
    • 'father' is located at top left corner
    • 'name' should be located at the center bottom cell
  • 'Female' items:
    • should contain red line at right
    • Text field in the center of cell which contains list of children with every even child has white font
    • In the end of children list should append 'name' with bold style
  • then user press 'Update' button, multichoosing dialog will be shown
  • There are 3 checkbox items at this dialog: Male, Female and Child
  • After closing dialog list should be updated
  • Update data should be downloaded every time with json-format
  • endpoint is located here
  • Request is HTTP Get request with syntax /api?types=male|female|child
  • If you want to get items with only 'child' type, you should use /api?types=female
  • If you have any question which is not clarified with this requirements, please make your own decision.
  • After application would be created, please pack sources to zip archive and send it to mail: mali@ciklum.com
